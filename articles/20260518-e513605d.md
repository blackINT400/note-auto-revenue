## なぜ副業エンジニアの9割はAIツール導入で情報漏洩リスクを「見ていない」のか

正直に言う。

Claude Codeを副業案件のコードレビューに使い始めたとき、最初の3日間、自分が何をリスクにさらしていたか、まったく気づいていなかった。

クライアントのコードをそのままプロンプトに貼り付けた。データベースのスキーマも、APIキーの命名規則も、一部の設定ファイルも。「AIに聞いたほうが早い」という判断だけで動いていた。

気づいたきっかけは些細なことだった。契約書の「秘密保持条項」を読み返したとき、そこに「第三者のサービスへの情報提供を禁ずる」という一文があった。

Anthropicのサーバーに送信したデータは、法的には「第三者への提供」に該当しうる。

副業で月20〜40万円を稼いでいる人間が、一つのミスで契約解除と損害賠償リクエストを同時に受け取る可能性がある。これは感情の話ではなく、構造上そうなっている。

---

## AIツールとNDAの交差点を誰も教えてくれない

副業市場でAI活用が加速している。2024年のランサーズ調査では、フリーランスエンジニアの68%が「業務にAIツールを使っている」と回答している。

一方で、NDA（秘密保持契約）の内容をきちんと読んでいる副業エンジニアは体感でいうとかなり少ない。

契約書は難しいし、長いし、案件を取った興奮の中で「まあ大丈夫だろう」と処理される。それが普通だ。

だが構造上、副業エンジニアには会社員より高いリスクが乗っている。

会社員ならば「社内でのAIツール利用ガイドライン違反」は人事案件だ。最悪でも始末書か降格。しかし個人事業主（副業含む）の場合、損害賠償は個人に直撃する。法人の盾がない。

Claude Codeは優秀なツールだ。コードを書く速度が体感で2〜3倍になる。しかしそれは、ミスをしたときの速度も2〜3倍になるということでもある。

---

## 「ガードレール」という概念の正体

ガードレールという言葉は、道路の話をしているわけではない。

FXをやっていた頃に学んだことがある。勝ちトレードよりも「負けないための構造設計」のほうが、長期的な収益を決定する。ルールを破ったときに何が起きるかを事前に設計しておくことで、感情が動いても行動の限界値が変わらない。

AIツールの社内導入（あるいは副業現場への個人導入）も同じ構造だ。

ガードレールとは「ミスをしない意志力」ではなく、「ミスをしても被害が最小化される物理的な設計」のことだ。意志力は使うものではなく、使わなくていい仕組みに変換するものだ。

以下、実際に自分が設定した5項目を書く。

ここからが本題だ。

## 有料パート：機密情報を漏らさないための設定5項目

### 項目1：`.claude/settings.json` でプロジェクトスコープを明示的に制限する

Claude Codeはデフォルトでは、起動したディレクトリ以下のファイルにアクセスできる。これは便利だが、副業案件では危険だ。

`~/projects/client_a/` の中に `.env` ファイルが含まれているとき、Claude Codeはそのファイルを文脈として読み込む可能性がある。`.env` にはAPIキー、DBのパスワード、外部サービスの認証情報が含まれていることが多い。

設定ファイルで明示的に「読んでいいファイル」を制限する。

```json
{
  "permissions": {
    "allow": [
      "Read(src/**)",
      "Read(tests/**)"
    ],
    "deny": [
      "Read(.env*)",
      "Read(*.pem)",
      "Read(config/secrets*)",
      "Read(**/*secret*)",
      "Read(**/*credential*)"
    ]
  }
}
```

`deny` のパターンは最初に広めに設定しておく。後から緩めるのは簡単だが、後から締めるのは「すでに送信した後」になる。

実際にこの設定を入れてから、誤ってシークレット系ファイルをコンテキストに含めたことは一度もない。

---

### 項目2：`CLAUDE.md` に「禁則事項」を明文化する

`CLAUDE.md` はClaude Codeが自動的に読み込むプロジェクト固有の指示ファイルだ。

ここに、AIへの指示としてではなく「自分への確認チェックリスト」を書く。

```markdown
# このプロジェクトのルール

## 絶対に含めないもの
- 本番環境のAPIキー・シークレット
- クライアント名（コード内では `CLIENT_X` で統一）
- 個人情報（メールアドレス、氏名、住所）
- 本番DBのスキーマ全体（テーブル名だけなら可）

## 質問するときの形式
- コードは必ず匿名化してから貼る
- エラーメッセージに含まれるパスはローカルパスのみ（本番サーバーパスは除く）
```

これは人間のためのドキュメントだ。AIが読む前に、自分が読む。

副業エンジニアが陥りやすいのは「コピペの速度が上がりすぎて、確認のステップが消える」という状態だ。`CLAUDE.md` は、その確認ステップを強制的に存在させる仕掛けだ。

---

### 項目3：APIキーは環境変数経由・直接ハードコードをCIで検知する

Claude Code自体の話ではなく、副業コードを書く際の話だ。

AIを使って速く書けるようになると、「とりあえず動くコード」をそのままコミットする速度も上がる。ハードコードされた認証情報がGitHubのパブリックリポジトリに上がった事例は、2023年だけで数千件以上（GitGuardianレポート）報告されている。

副業案件でパブリックリポジトリを使うことは少ないが、誤った設定でパブリックになるリスクはある。

`pre-commit` フックと `gitleaks` を組み合わせる。

```bash
# gitleaksのインストール
brew install gitleaks

# pre-commitフックの設定
cat > .git/hooks/pre-commit << 'EOF'
#!/bin/sh
gitleaks detect --staged -v
if [ $? -ne 0 ]; then
  echo "シークレット検出: コミットを中止"
  exit 1
fi
EOF
chmod +x .git/hooks/pre-commit
```

これで「コミット前に必ずシークレットスキャンが走る」状態になる。速く書けるようになった分、このブレーキが必要だ。

---

### 項目4：Enterprise契約とTeam契約の違いを契約前に確認する

ここが最も見落とされている点だ。

Anthropicのデータ保持ポリシーは、契約プランによって異なる。

- **無料・Proプラン**：送信したデータはモデル改善に使われる可能性がある（デフォルト設定）
- **Claude for Work（Team）**：データはトレーニングに使用されない（2024年現在）
- **Enterprise契約**：より強固なデータ隔離・監査ログが提供される

副業エンジニアが個人でProプランを使ってクライアントのコードを送信している場合、Anthropicの利用規約上「データはトレーニングに使用されうる」状態になっている。

NDAに「第三者サービスへの機密情報提供を禁ずる」条項があり、かつデータがトレーニングに使われうる場合、法的グレーゾーンに踏み込む。

対策は単純だ。月額$25（2024年価格）のClaude for Workに切り替え、オプトアウト設定を確認する。

副業収入が月20万円ある場合、月2,500円の保険料として考えれば、コストパフォーマンスは明らかだ。契約解除リスクと比較すれば、1000分の1以下のコストだ。

### 項目5：副業収入とAIツール費用の経費計上を正確に行う

ここで副業と税務の話に接続する。

Claude for WorkやClaude Codeの費用は、副業収入に対する必要経費として計上できる。これは構造上そうなっている。

確定申告（青色申告）の場合、必要経費として計上できるもの：

- Claude for Work月額費用（副業に使用している割合分）
- GitHub Copilotなど他のAIツール費用
- 副業専用のPC・機材（按分計算）

具体的な計算例を出す。

年間Claude for Work費用：$25 × 12 = $300 ≈ 45,000円（為替レート次第）

副業収入が年間240万円（月20万円）の場合、所得税率は20%前後。

45,000円の経費計上で、節税額は約9,000円。大きくはないが、「AIツールを使うほど節税になる」という構造は知っておいて損はない。

さらに、副業で使うクラウドサービス（AWS、Vercel等）、ドメイン費用、技術書籍費用を合算すると、年間10〜20万円の経費計上は一般的なエンジニアでも現実的な範囲に入る。

freeeの青色申告プランを使っている場合、各費用の勘定科目は「通信費」または「消耗品費」で計上するのが一般的だ。AIツール費用は「通信費」か「外注費」かで迷う人が多いが、SaaS型ツールは「通信費」で問題ない（確認は税理士に委ねること）。

## ガードレールは「制限」ではなく「速度の担保」だ

5項目を並べると、制限のリストに見える。

だが本質は逆だ。

これらの設定を完了させた状態でClaude Codeを使うと、思考が「これ送っていいかな」という確認作業から完全に解放される。ガードレールが存在することで、アクセルを全開にできる。

FXで学んだ構造と同じだ。リスク管理が甘いトレーダーは、常にポジションが怖い。リスク管理が完璧なトレーダーは、ポジションを持つことを怖がらない。

副業エンジニアにとって、AIツールは「速度を上げる道具」だ。だがその道具を正しく使うためには、速度を上げた先に何があるかを設計しておく必要がある。

5項目のセットアップにかかる時間は、一度やれば1時間以内だ。その1時間が、副業収入を守る構造になる。

既に持っているスキルと速度がある。あとは、それを正しい構造に乗せるだけだ。
